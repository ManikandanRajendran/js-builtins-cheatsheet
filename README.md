## Features

- **Tabbed examples** ‚Äî 5‚Äì6 real, runnable code examples per function
- **LeetCode integration** ‚Äî 5 curated problems per function with difficulty badges (Easy / Medium / Hard) and a one-line hint explaining which concept to apply

## Key interview concepts covered

- **Mutation vs immutability** ‚Äî which methods mutate (`sort`, `reverse`, `splice`) vs return new arrays (`map`, `filter`, `slice`)
- **`map` vs `forEach`** ‚Äî when to use each and why `forEach` always returns `undefined`
- **`find` vs `filter`** ‚Äî early exit vs full scan
- **`reduce` as the universal tool** ‚Äî how it subsumes map, filter, and more
- **`Set` for O(1) deduplication** ‚Äî the `[...new Set(arr)]` pattern
- **`Map` over plain objects** ‚Äî when any-type keys, `.size`, and insertion order matter
- **`Number.isNaN` vs global `isNaN`** ‚Äî why strict checking matters
